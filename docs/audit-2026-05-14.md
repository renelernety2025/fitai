## Tier 2 — Customer-facing bugs (broken happy paths)

### T2.1 Web + mobile register **always fails 400**
`apps/web/src/app/(auth)/register/page.tsx:36` + `apps/mobile/src/screens/RegisterScreen.tsx:29` send `level` field; `RegisterDto` doesn't declare it; `forbidNonWhitelisted` rejects. **New user signup is fully broken in production.**

### T2.2 Admin pages permanently inaccessible
`AuthService.toProfile()` (line ~125-134) strips `isAdmin`. `/auth/me` never returns it. Web admin checks `user.isAdmin === true` → always false.

### T2.3 `/share/[sessionId]` returns 401
`GymSessionsController.getShareCard` has `@UseGuards(JwtAuthGuard)`, but share page sends no Authorization header. Every shared workout link 401s for recipients.

### T2.4 Clips have NO video playback (web + mobile)
`/clips` page renders only caption text. `ClipsScreen` same. Backend stores `s3Key` (`clips/{userId}/{id}.mp4`) but FE discards it. No HLS pipeline, no signed-read URL endpoint.

### T2.5 Post photos broken in production
`PostCard.tsx:89` builds `src={process.env.NEXT_PUBLIC_CDN_URL || ''}/${s3Key}` — env not set → 404. Mobile expects `photos.slice(0,4)` as URL strings but BE returns `[{s3Key, order}]` objects → blank Image.

### T2.6 Hashtag `postCount` never increments for new tags
`posts.service.ts:225` upsert: `create: { name }` (no `postCount: 1`). Fresh tags stuck at 0 → `getTrendingHashtags` orders by `postCount desc` → trending stays empty.

### T2.7 Duels can't be created from UI
`/duels` page CTA `disabled` with "Challenge (coming soon)". Backend `POST /api/duels/challenge` exists. Feature is shipped backend-only.

### T2.8 Boss-fight complete DTO mismatch
`CompleteBossDto` only accepts `{score, defeated}`. Service comparison against `targetTime` requires elapsed time but DTO rejects `durationSec` with 400 (forbidNonWhitelisted). Feature broken.

### T2.9 Stale active season
`/seasons/current` returns "Jarni vyzva 2026" with `endDate: 2026-04-30`. Today is 2026-05-14. No rollover cron. Users earn XP into expired season.

### T2.10 Streaks page contract mismatch
Live API returns `{available, maxPerMonth, usedDates}`, page reads `{available, max}`. Also no "Use freeze" CTA — `POST /streak-freeze/use` never called.

### T2.11 Onboarding not enforced + web skips fitness-test
`(app)/layout.tsx` doesn't check `getOnboardingStatus()`. New users land on empty dashboard. Web onboarding wizard never calls `submitFitnessTest` (mobile does).

### T2.12 Mobile no Forgot Password screen
LoginScreen has no link, no screen registered.

## Tier 3 — Security / IDOR / Throttle gaps

### T3.1 `workout-plans/:id` IDOR
`findOne` has no ownership check. Logged-in users can read any other user's private plan by UUID.

### T3.2 `home-training/*` fully public + no throttle
3 endpoints (quick/home/travel) have no `@UseGuards`, no `@Throttle`. Anyone on internet hits 4 KB workout JSON at unlimited rate.

### T3.3 `GET /api/exercises` no throttle (114 KB response, public)
Public catalog egress vector.

### T3.4 `BuddyProfileDto` accepts empty body
All fields `@IsOptional`. Live-tested: `POST /api/buddy/profile` with `{}` creates row with all nulls. No minimum-field requirement.

### T3.5 Various missing `@Throttle()`
- `POST /nutrition/photo-upload-url` (presigned URL spam)
- `POST /recipes/:id/photo-url`
- `POST /wearables/sync` (accepts 1000 entries)
- All gamification write endpoints (achievements/check, leagues/join, seasons/join, skill-tree/check, boss-fights/start|complete, daily-quests/complete)
- `PUT /users/me/password`, `DELETE /users/me` (credential ops, currently only global throttler)

### T3.6 `admin/verify-user/:id` 500 on missing user
Prisma `P2025` not caught → leaks 500. Should be 404.
